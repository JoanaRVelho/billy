Development Contribution

Instructions on how to contribute through development and how to make pull and merge requests

For issues with the Billy application you should submit your bug report to the Billy Issue tracker

Rules to bug reporting

  • 1. You should check if the bug has already been reported, to avoid duplicated issues, and check if the problem has already been solved.
  • 2. The issue must be labeled.
  • 3. Create an isolated and reproducible test case, where shows the steps that led to the problem occur.
  • 4. Share as much information as possible, like your operating system and version, the version of Billy, and other information that you think important.

Contribute

If you want to contribute to the project development you can download Billy from Billy's GitHub (to download Billy you must have to be registered at Git)
There are two ways to get Billy:

  1. Download the latest stable release,as a zip file from Billy's GitHub, and unzip to the folder that you want
  2. Or you can clone the repository through the command line:
$ git clone https://github.com/premium-minds/billy.git

Install Billy

To install Billy go to each billy's folder(core, jpa, gin and platypus) and execute the following steps:

  1. Enter in the billy-<project> folder:
    $ cd billy-<project>
  2. Compile, execute the tests and install billy-<project> in a local repository.
    • 2.1 Build the project: 
      $ mvn compile
    • 2.2 Compile and run the project tests: 
      $ mvn test
    • 2.3 Build the project jar: 
      $ mvn package
    • 2.4 Install the project jar in a local repository: 
      $ mvn install
